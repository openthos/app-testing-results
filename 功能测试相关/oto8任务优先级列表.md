| 优先级             | 含义                                                         |
| ------------------ | ------------------------------------------------------------ |
| Immediate 马上解决 | 问题必须马上解决，否则系统根本无法达到预定的需求。           |
| Urgent 急需解决    | 问题的修复很紧要，很急迫，关系到系统的主要功能模块能否正常。 |
| High 高度重视      | 有时间就要马上解决，否则系统偏离需求较大或预定功能不能正常实现。 |
| Normal 正常处理    | 进入处理队列，问题不影响需求的实现，但是影响其他方面的使用。 |
| Low 低优先级       | 问题在系统发布以前必须确认解决或确认可以不予解决。           |
